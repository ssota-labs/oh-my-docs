## Vision

Teams should be able to understand why a product exists, what a change promises,
and how it will be delivered without reconstructing intent from tickets, chats,
and code history.

Oh My Docs makes documentation the working surface for product development. A
small, linked set of durable artifacts becomes readable in a browser, available
as Markdown to agents, and verifiable as a graph.

## Principles

1. Intent precedes implementation.
2. Stable IDs connect artifacts without coupling them to filenames.
3. Human review decides product quality; automation checks structural integrity.
4. Documentation and agent-readable Markdown are two views of the same source.
5. The foundation stays product-neutral so teams can add their own domain.

## Not yet in scope

Phase 1 does not provide shared UI packages, a visual planning canvas,
authentication, persistence services, or project-management integrations.
