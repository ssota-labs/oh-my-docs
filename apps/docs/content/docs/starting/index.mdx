## Understand the product

Read the [product vision](/docs/vision), then browse active
[requirements](/docs/planning/prds) and [user stories](/docs/planning/stories).
Use the [Domain catalogs](/docs/domain) for shared terms, models, and policies.

## Plan a change

Follow the [planning workflow](/docs/workflow/planning): discover candidate user
stories, moderate the product decisions, then maintain living Domain and typed
[specification](/docs/spec) contracts (data model, system model, CLI). Durable
technical choices belong in [architecture decisions](/docs/adr).

## Implement reviewed work

Use the [development contract](/docs/workflow/development) and select a ready
[implementation plan](/docs/plans). Coding agents should also read the
[agent guide](/docs/agent).
