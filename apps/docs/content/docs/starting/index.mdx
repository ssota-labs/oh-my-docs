## Understand the product

Read the [product vision](/docs/vision), then browse active
[requirements](/docs/planning/prds) and [user stories](/docs/planning/stories).

## Plan a change

Follow the [planning workflow](/docs/workflow/planning). Observable behavior belongs
in typed [specifications](/docs/spec) (data model, system model, CLI), and durable
technical choices belong in [architecture decisions](/docs/development/adr).

## Implement reviewed work

Use the [development contract](/docs/workflow/development) and select a ready
[implementation plan](/docs/development/plans). Coding agents should also read the
[agent guide](/docs/agent).
