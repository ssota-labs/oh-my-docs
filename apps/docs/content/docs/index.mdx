Oh My Docs keeps product intent, user outcomes, observable contracts, decisions,
and implementation plans in one navigable system.

## Start here

- Read the [product vision](/docs/vision).
- Follow the [planning workflow](/docs/workflow/planning).
- Use the [development contract](/docs/workflow/development) before changing code.
- Browse the first [PRD](/docs/planning/prds/prd-docs-foundation) and its linked graph.

Every page is also available as processed Markdown by appending `.md` to its
docs URL, for example `/docs/vision.md`.
