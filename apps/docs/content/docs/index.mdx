Oh My Docs keeps product intent, user outcomes, observable contracts, decisions,
and implementation plans in one navigable system.

## Start here

- Use the [starting guide](/docs/starting) to choose the right path.
- Read the [product vision](/docs/vision).
- Share product language through [Domain terms, models, and policies](/docs/domain).
- Follow the [planning workflow](/docs/workflow/planning).
- Use the [development contract](/docs/workflow/development) before changing code.
- Give coding agents the [CLI / skill runtime](/docs/spec/cli).
- Browse the foundation [PRD](/docs/planning/prds/prd-docs-foundation) and the
  skill/UI refresh [PRD](/docs/planning/prds/prd-skill-and-ui-refresh-loop).
- Review the [PDCA and task orchestration PRD](/docs/planning/prds/prd-pdca-domain-and-task-orchestration).

Every page is also available as processed Markdown by appending `.md` to its
docs URL, for example `/docs/vision.md`.
