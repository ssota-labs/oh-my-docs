## Scope

Observable navigation and rendering contracts for the handbook app and the
default skill template IA. Does not define `.omd` task execution.

## Routes

- `/` redirects to `/docs` (handbook entry).
- `/docs` and `/docs/<slug>` render handbook pages.
- `/docs/<slug>.md` returns the corresponding processed Markdown with
  `text/markdown; charset=utf-8`.
- Unknown page slugs return not found.

## Default information architecture

Top-level sidebar order:

`Home Ôćĺ Vision Ôćĺ Start here Ôćĺ Workflow Ôćĺ Planning Ôćĺ Plans Ôćĺ ADR Ôćĺ Spec`

| Section | Sidebar behavior |
|---|---|
| Planning | Folder with index-only catalogs: PRDs, Stories |
| Plans | Index-only catalog at the top level |
| ADR | Index-only catalog at the top level |
| Spec | Folder of typed living pages (children visible) |

There is no intermediate Development section in the default IA.

## Index-only catalogs

Configured catalog folders keep detail pages registered in `meta.json` for
validation, but the page tree exposes only the catalog index. Opening the
index shows a table of documents (ID, status when present, title). Detail pages
provide back-to-index and sibling previous/next links.

Default catalog prefixes:

- `planning/prds` Ôćĺ `/docs/planning/prds`
- `planning/stories` Ôćĺ `/docs/planning/stories`
- `plans` Ôćĺ `/docs/plans`
- `adr` Ôćĺ `/docs/adr`

## Spec typed boundaries

Spec is not a flat `SPEC-*` catalog. The sidebar lists a stable set of
boundaries:

- [Data model](/docs/spec/data-model)
- [System model](/docs/spec/system-model)
- [CLI](/docs/spec/cli)

**Data model** and **System model** are index-only folders (same mechanism as
PRDs): the sidebar shows the boundary entry; detail pages live under the folder
and appear in the boundary index table, not in the sidebar. CLI may stay a
single page until it needs children.

Initiative-specific behavior belongs under these boundaries (as hidden child
pages or sections), not as an unbounded top-level Spec list.

## Source

Content lives under the configured docs content root (this repo:
`apps/docs/content/docs`). Fumadocs validates frontmatter and generates the
page tree. Skill `adopt` / `sync` keep template `meta.json` and catalog wiring
aligned with `.omd` IA defaults.

## Acceptance criteria

- Catalog detail pages do not appear as sidebar children.
- Catalog indexes render tables, not long unstructured link stacks alone.
- ADR and Plans are reachable as second-level sidebar items.
- Spec sidebar shows only the boundary entries; Data model and System model
  children stay index-only like PRDs.
