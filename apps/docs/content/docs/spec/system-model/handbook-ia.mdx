## Scope

Observable navigation and rendering contracts for the handbook app and the
default skill template IA.

## Routes

- `/` redirects to `/docs` (handbook entry).
- `/docs` and `/docs/<slug>` render handbook pages.
- `/docs/<slug>.md` returns the corresponding processed Markdown with
  `text/markdown; charset=utf-8`.
- Unknown page slugs return not found.

## Default information architecture

`Home → Vision → Start here → Domain → Workflow → Planning → Plans → ADR → Spec`

| Section | Sidebar behavior |
|---|---|
| Vision / Start here | Single pages (not folders) |
| Domain | Folder with index-only catalogs: Glossary, Models, Policies |
| Planning | Folder with index-only catalogs: PRDs, Stories |
| Plans | Index-only catalog at the top level |
| ADR | Index-only catalog at the top level |
| Spec | Folder of boundary entries; Data model and System model are index-only |

There is no intermediate Development section and no default Agent section.

## Index-only catalogs

Configured catalog folders keep detail pages registered in `meta.json` for
validation, but the page tree exposes only the catalog index. Opening the
index shows a table of documents. Detail pages provide back-to-index and
sibling previous/next links.

Default catalog prefixes:

- `domain/glossary` → `/docs/domain/glossary`
- `domain/models` → `/docs/domain/models`
- `domain/policies` → `/docs/domain/policies`
- `planning/prds` → `/docs/planning/prds`
- `planning/stories` → `/docs/planning/stories`
- `plans` → `/docs/plans`
- `adr` → `/docs/adr`
- `spec/data-model` → `/docs/spec/data-model`
- `spec/system-model` → `/docs/spec/system-model`

## Spec boundaries

Sidebar lists only:

- [Data model](/docs/spec/data-model)
- [System model](/docs/spec/system-model)
- [CLI](/docs/spec/cli)

Data model and System model are index-only folders (same mechanism as PRDs).
CLI may remain a single page until it needs children.

## Acceptance criteria

- Catalog and Spec-boundary detail pages do not appear as sidebar children.
- Domain sidebar shows only Glossary, Models, and Policies (indexes).
- Spec sidebar shows only the three boundary entries.
- ADR and Plans remain second-level sidebar items.
