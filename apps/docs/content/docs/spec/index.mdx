Specifications are typed boundary folders/pages. Data model and System model
behave like PRDs: open the boundary for an index table; children stay out of
the sidebar.

| Boundary | Role |
|---|---|
| [Data model](/docs/spec/data-model) | Catalog IDs and `.omd` tasks |
| [System model](/docs/spec/system-model) | IA, PDCA, orchestration, task board |
| [CLI](/docs/spec/cli) | Skill install and `omd.mjs` actions |

Workflow gates for implementation remain in
[Development contract](/docs/workflow/development).
