## Scope

Convert Oh My Docs from public npm CLI packages to a single `skills/oh-my-doc`
workflow skill. Agents use `inspect` / `adopt` / `new` / `check` / `sync`. Persist
project state in `.omd/`. Keep Fumadocs shell packages on npm; copy planning UI
vocabulary from the skill template on `adopt`. Mirror the skill into Cursor,
Codex, and Claude Code plugin wrappers. Remove `packages/cli`, `packages/core`,
and `packages/create-oh-my-docs`.

## Implementation sequence

1. Land canonical skill runtime, schemas, templates, and tests.
2. Wire CI checks to skill scripts (`check:planning`, `check:ui-snapshot`,
   `check:skills`, `check:docs-first`, `check:release-version`).
3. Sync plugin skill mirrors from `skills/oh-my-doc`.
4. Drop public CLI packages and obsolete setup skills.
5. Document bootstrap behavior while the skill-era docs-first gate is introduced.

## Verification

- `pnpm test`
- `pnpm typecheck`
- `pnpm build`
- `pnpm check:planning`
- `pnpm check:ui-snapshot`
- `pnpm check:skills`
- `pnpm check:release-version`
- Docs-first: skill-era gate absent on base → bootstrap skip; present on later
  bases → require `Plan:` citing a ready|active plan on that base.

## Boundaries

No public npm CLI publish surface. No shadcn registry. Dogfood `@oh-my-docs/ui`
remains private.

## Rollback

Revert to the foundation commit that still ships packages/core and the TypeScript
docs-first gate wrapper.
