## Scope

1. Rewrite agent and planning handbook pages for skill-first install/`omd.mjs`.
2. Extend `inspect` / `check` (and supporting runtime helpers) so they report
   missing or stale UI vocabulary files relative to
   `skills/oh-my-doc/templates/default/packages/ui` (or the installed skill's
   template path).
3. Keep `adopt` / `adopt --force` as the refresh mechanism; do not add a
   registry.
4. Sync plugin skill mirrors after runtime changes (`pnpm sync:skills`).
5. Add or extend runtime tests for snapshot health reporting.

## Implementation sequence

1. Land this planning set and handbook bootstrap rewrites (docs-only).
2. Add a reusable UI snapshot comparison helper used by `inspect` and `check`.
3. Surface structured fields in JSON (`missing`, `stale`, `identical`,
   `recommended: adopt | adopt --force`).
4. Extend runtime tests; run `pnpm sync:skills` and `pnpm check:ui-snapshot`.
5. Mark this plan `done` after verification.

## Verification

- `pnpm check:planning`
- `pnpm check:ui-snapshot`
- `pnpm check:skills`
- `node --test skills/oh-my-doc/tests/*.test.mjs`

## Migration and compatibility

Existing projects without `.omd` still need `adopt` first. Projects with local
UI edits remain protected until `--force`.
