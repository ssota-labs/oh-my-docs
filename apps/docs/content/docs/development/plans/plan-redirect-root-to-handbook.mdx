## Scope

1. Change `SPEC-docs-foundation` so `/` redirects to `/docs` (docs-only in this PR).
2. Replace `apps/docs/app/page.tsx` with a Next.js redirect to `/docs`.
3. Keep the skill scaffold template (`templates/default/docs/app/page.tsx`) aligned.
4. Sync plugin skill mirrors after template changes.

## Implementation sequence

1. Merge the specification and this ready plan.
2. Implement permanent redirect at `/` in the dogfood docs app.
3. Mirror the same redirect in the skill default template; `pnpm sync:skills`.
4. Smoke the docs app: `/` → `/docs`.

## Verification

- `pnpm check:planning`
- `pnpm check:skills` (after template sync)
- Manual or automated hit of `/` returns redirect to `/docs`

Bookmarks to `/` land on the handbook home. The previous intro copy on `/` is
removed; handbook `index` remains the landing content.
