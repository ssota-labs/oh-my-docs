As a contributor using a coding agent, I want to know when the project's
planning UI snapshot is missing or behind the installed `oh-my-doc` skill so
that I can refresh `packages/ui` without hunting through templates by hand.

## Context

Adopt copies UI vocabulary from the skill into the project. Skill updates do
not automatically rewrite that copy. Contributors need a clear detect → refresh
loop.

## Acceptance criteria

- `inspect` or `check` reports when the UI package is missing.
- `inspect` or `check` reports when project UI files differ from the skill
  template snapshot (stale or locally modified).
- Recommended next action is `adopt` to create missing files, or
  `adopt --force` to overwrite divergent skill-managed UI files.
- Handbook agent and planning pages describe this loop without referring to a
  public CLI or a `setup` skill.

## Out of scope

- Merging local UI customizations with upstream template changes automatically.
- Updating Fumadocs npm packages as part of skill refresh.
