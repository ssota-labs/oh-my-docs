As a contributor, I want to move from a product requirement to its user outcome,
contract, decision, and implementation plan so that I can understand a change
before editing code.

## Acceptance criteria

- The handbook navigation exposes each artifact catalog.
- Each artifact has a stable ID and links to related context.
- Broken IDs, references, or catalog registrations fail validation.
- Appending `.md` to a docs URL returns processed Markdown.
